Problem: Nav items wrap into two lines and vertical alignment is off.
Goal: One clean row on desktop (≥1024px), logo at left, links centered with equal spacing, Donate + Theme toggle at right. On smaller screens, collapse to a hamburger menu (no wrapping).

Requirements
  • Header remains sticky, glass/liquid style intact.
  • Desktop: no wrapping; consistent vertical centering; equal horizontal gaps.
  • If the viewport can’t fit all links, switch to mobile/hamburger instead of wrapping.
  • Maintain WCAG AA contrast for all states.

3) JS — open/close mobile (prevents wrap instead of shrinking)

const btn = document.getElementById('menuButton');
const panel = document.getElementById('mobilePanel');
btn?.addEventListener('click', () => {
  const isHidden = panel.hasAttribute('hidden');
  if (isHidden) panel.removeAttribute('hidden'); else panel.setAttribute('hidden','');
  btn.setAttribute('aria-expanded', String(isHidden));
});

4) Notes (why this fixes alignment)
  • No wrapping: .nav-list { white-space: nowrap; overflow: hidden; }
  • Single row: center block (.primary-nav) flexes between brand and actions; links have equal gap.
  • Breakpoint first: when there isn’t enough width, we hide the list and show burger — no two-line mess.
  • Vertical centering: every interactive element uses a fixed height (36px) or line-height 1 within a 72px bar.
  • Brand doesn’t wrap (white-space: nowrap) so the logo/text never splits.

Acceptance Criteria
  • On ≥1200px, header shows one row only, centered links, no wrapping at any zoom level.
  • On <1200px, links collapse to hamburger; desktop never shows two lines.
  • Donate button and theme toggle stay right-aligned and vertically centered.
  • Glass/liquid styling unchanged; contrast passes AA.
