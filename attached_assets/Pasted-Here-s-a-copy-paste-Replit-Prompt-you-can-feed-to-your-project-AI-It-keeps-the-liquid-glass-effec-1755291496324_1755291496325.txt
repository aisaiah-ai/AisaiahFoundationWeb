Replit Prompt — Improve Top Navigation (Keep Liquid Effect)

Project: AIsaiah Foundation web app (dark theme).
Goal: Keep the existing “liquid / glass” header vibe, but make the top menu highly readable and usable across desktop + mobile.

What to change
  1.  Header container
  • Keep a translucent, liquid/glass style but raise contrast for the menu text.
  • Use position: sticky; top: 0; z-index: 50; so the bar stays visible on scroll.
  • Apply a subtle backdrop blur and darker overlay so text passes WCAG AA on dark backgrounds.
  • Add a soft bottom border or shadow to separate the header from content.
  2.  Logo + Brand
  • Left-align logo, maintain current SVG.
  • Ensure the brand name AIsaiah Foundation has minimum contrast ratio 4.5:1 over the header.
  3.  Nav items (Desktop ≥1024px)
  • Increase font weight to 600, letter-spacing +0.2px.
  • Default color: #E6ECF5 (or rgba(230,236,245,.92)).
  • Hover/active: text to #FFFFFF and add a liquid underline: a 2px rounded bar that grows from center to full width (200ms).
  • Focus state (keyboard): 2px focus ring #67E8F9 outside text.
  4.  Dark/Light toggle
  • Keep the moon icon, but ensure it has a visible focus ring and tooltip title="Toggle theme".
  5.  Mobile nav (≤1023px)
  • Replace the cramped top links with a right-aligned hamburger.
  • Slide-down panel with the same glass/liquid look (blur + tint), full-width, safe-area insets respected.
  • Each item is at least 44px tall; include separators at 10% opacity.
  6.  Liquid effect (retain)
  • Keep the gooey/liquid background in the header using CSS/SVG filter (no heavy canvas).
  • Tone it down 30–40% so it doesn’t lower text contrast. Mask it behind the overlay layer.
  7.  Accessibility
  • All text in header passes WCAG AA (≥4.5:1).
  • Full keyboard navigation (Tab → Shift+Tab), aria-expanded on hamburger, aria-label on toggle.
  • Reduce motion preference: if prefers-reduced-motion, disable underline animation.
  8.  Performance
  • Use will-change: backdrop-filter and avoid large box-shadow blurs.
  • Only mount the mobile panel in DOM when open.

Implement with Tailwind (preferred) or vanilla CSS

If Tailwind is available, add these utilities; otherwise, create equivalent CSS.

  @apply text-white;

Minimal JS (vanilla)

const btn = document.getElementById('menuButton');
const panel = document.getElementById('mobilePanel');

btn?.addEventListener('click', () => {
  const open = panel.classList.toggle('hidden') === false;
  btn.setAttribute('aria-expanded', String(open));
});

// Reduce motion: disable underline animation
if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  document.documentElement.style.setProperty('--tw-duration', '0ms');
}

Liquid / Gooey filter tip
  • Keep your existing SVG filter (e.g., feGaussianBlur + feColorMatrix + feBlend) but wrap it under the header using an absolutely positioned SVG, and lower the shapes’ opacity to 0.25–0.35.
  • Put the tint/overlay above the gooey layer: a ::before on the header with background: rgba(10,12,18,0.55); backdrop-filter: blur(12px);.

Acceptance criteria
  • Nav text contrast ≥ 4.5:1 on all header states.
  • Sticky header with clear separation from content.
  • Smooth liquid vibe remains visible but subtle.
  • Keyboard-navigable; focus styles visible.
  • Mobile menu is readable, touch-friendly (44px min).
  • No layout shift; Lighthouse Accessibility ≥ 95 for header section.
